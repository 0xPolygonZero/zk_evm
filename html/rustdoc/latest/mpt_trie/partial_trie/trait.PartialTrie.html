fn delete<K>(&mut self, k: K) -> TrieOpResult<Option<Vec<u8>>>
where K: Into<Nibbles>,

Deletes a Leaf node or Branch value field if it exists.

To agree with Ethereum specs, deleting nodes does not result in the trie removing nodes that are redundant after deletion. For example, a Branch node that is completely empty after all of its children are deleted is not pruned. Also note:

  • Deleted leaves are replaced with Empty nodes.
  • Deleted branch values are replaced with empty Vecs.
Â§Panics

If a Hash node is traversed, a panic will occur. Since Hash nodes are meant for parts of the trie that are not relevant, traversing one means that a Hash node was created that potentially should not have been.
